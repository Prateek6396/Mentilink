## 1. Overview

Mentilink is a web-based mentoring platform designed to facilitate mentor-student interactions, scheduling, and session management for career and skill development. The system supports multiple user roles, real-time scheduling, session summaries, notifications, and reporting functionalities.

---

## 2. System Architecture & Components

### 2.1 System Architecture

- **Client-Server Model**: Web application with a frontend interface and backend processing.
- **Frontend**: Responsive web interface compatible with desktops, tablets, and smartphones.
- **Backend**: RESTful API server handling business logic, data processing, and integrations.
- **Database**: Centralized database for storing user data, session info, schedules, summaries, and logs.
- **Hosting & Deployment**:
  - Development: Localhost
  - Deployment: Heroku, PythonAnywhere, or similar cloud services
- **Version Control**: GitHub for code collaboration and backup

### 2.2 Technologies & Frameworks

| Layer | Technology / Framework | Details |
|---------|-------------------------|---------|
| Backend | Python + Flask | Web framework for API development |
| Frontend | HTML, CSS, Bootstrap 5 (for responsive design), JavaScript | UI development with responsiveness and interactivity |
| Database | SQLite (initial), scalable to MySQL/PostgreSQL | Data storage for users, sessions, summaries, etc. |
| Email | SMTP via Gmail or SendGrid | Email notifications and reminders |
| PDF Export | ReportLab or XHTML2PDF | Generate PDF reports and summaries |
| Version Control | Git & GitHub | Code management and collaboration |

## 3. Functional Requirements

### 3.1 User Roles & Authentication

- **Mentor**
  - Register/Login
  - Create, edit, delete available time slots
  - View and manage booked sessions
  - Submit session summaries
  - Rate/review mentees post-session
  - Export session summaries as PDF
  - View and update profile
- **Student**
  - Register/Login
  - View available mentor slots
  - Book slots
  - Receive email reminders
  - Submit reviews post-session
  - View session summaries
  - Export summaries as PDF
- **Admin** (Optional)
  - Manage users
  - Manage slots
  - View system reports
  - Manage notifications

### 3.2 Core Features & Functionalities

#### 3.2.1 Authentication & Authorization
- Separate registration pages for mentors and students
- Secure password hashing (e.g., bcrypt)
- Role-based access control
- Login/Logout functionality
- Role-based dashboard views

#### 3.2.2 Slot Management
- Mentors can create, edit, delete time slots
- Slots are validated and stored in the database
- Slots are displayed in calendar or list format
- Students can view and book available slots
- System checks slot availability before booking

#### 3.2.3 Booking & Session Management
- Students can book a slot (one per student per time period)
- System updates slot status upon booking
- Mentors can view, edit, or delete booked slots
- Post-session, mentors submit summaries
- Students can view and mark issues as resolved
- Session summaries include topic, discussion, status (solved/pending)

#### 3.2.4 Notifications & Reminders
- Email notifications for booking confirmations, cancellations, and reminders
- In-app messages for booking status
- Optional email confirmation after successful booking
- Reminders sent before session start time

#### 3.2.5 Session Summaries & Reports
- Mentors submit summaries post-session
- Summaries include date, time, topics discussed, issues, resolutions
- Export summaries as PDF
- View summaries in dashboard
- Students can review and mark issues as resolved
- Mentors can review and rate students
- Export session summaries as PDF or email

#### 3.2.6 Rating & Feedback
- Students rate mentors (1-5 stars)
- Optional text feedback
- Average rating displayed on mentor profile
- Feedback stored for review

#### 3.2.7 Data Export & Review
- Export session summaries as PDF
- Review and rate mentors
- Downloadable reports

### 3.3 Additional Features
- Email reminders before sessions
- Automatic email notifications for booking, cancellations, and summaries
- Dashboard views for mentors and students
- Responsive design for mobile and tablets

## 4. Data Flow & System Processes

### 4.1 Data Entities & Relationships

| Entity | Attributes | Relationships |
|---------|--------------|--------------|
| User | user_id, name, email, password_hash, role (mentor/student/admin), profile info | One-to-many with Slots, Sessions, Summaries |
| Slot | slot_id, mentor_id, date, start_time, end_time, status (available/booked) | Belongs to Mentor, booked by Student |
| Booking | booking_id, slot_id, student_id, mentor_id, status | Links User and Slot |
| Session Summary | summary_id, booking_id, mentor_id, student_id, date, topics, issues, resolutions, rating | Belongs to Booking |
| Review | review_id, mentor_id, student_id, rating, comments | Belongs to Mentor and Student |

### 4.2 Data Flow Diagram (DFD)

- External entities: Mentor, Student, System
- Data stores: Users, Slots, Bookings, Summaries, Reviews
- Processes:
  - Authentication
  - Slot Management
  - Booking Handling
  - Session Summary Submission
  - Notifications & Reminders
  - Export & Reports
